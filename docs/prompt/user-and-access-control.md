给网站设计全套的权限管理：
- 数据库
    - 增加 user 表，至少包含
        - 含 id
        - email ()
        - name：用户设置的用户名
        - is_admin：标记是否是管理员，1是，0不是
        - 创建时间
        - 成功注册时间
    - pipeline 表增加归属用户id记录
- 网站需要登录才能使用(相关页面尽量用Ant design 既有的，且尽量设计得好看易用)
    - 没有登录/登录信息过期的用户自动跳登录对话框
        - 过期时间可以设置久一点（1个月）
    - 登录时需填 email
        - 填写后如果发现 email 不存在，则提示不存在
        - 如果 email 存在则向对应 email 发一封邮件，包含说明和随机生成的 4 位数字
        - 然后跳转到邮件验证页面，页面内提示已发送邮件，请填邮件内的4位数字，填写成功则成功登录
        - 
    - 登录对话框包含新用户注册
        - 注册时让用户输入 email
        - 输入用户名
        - 点击确定以后，也发送一封邮件，包含说明和随机生成的 4 位数字
        - 然后跳转到邮件验证页面，页面内提示已发送邮件，请填邮件内的4位数字，填写成功则完成注册并成功登录
    - 登录后页面右上角展示用户头像（生成一个 情报鸭相关的通用小头像用着），并且可以看到自己名字，点击后可以选择 个人设置/退出登录
        - 个人设置页面暂时仅用支持名字修改
        - 退出登录就点击以后退出登录并回到登录页面
    - 管理员权限
        - 可以如当前看/管理各个页面内容
        - 且增加一个"用户管理"(和"投递管理"那些并列)，页面内可以看到所有用户list,点击可以跳单用户详情页面
        - 投递管理可以看到所有用户的投递（增加一列展示投递的创建用户名（并且可以点击跳对应用户的详情页面））
    - 用户权限
        - 资讯管理可以都看到
        - 来源管理无法修改启动状态（但能看到），没有编辑和删除操作
        - 类别管理无法修改启动状态（但能看到），没有编辑和删除操作
        - AI评估无法修改启动状态（但能看到），没有编辑和删除操作
        - 投递管理可以看到和修改归属于自己的投递
